# Product

## Register

product

## Users

兩層使用者：

1. **介面模擬的角色**：產線品管工程師，一次丟一批 PCB 影像，需要快速看出哪些板子不合格、為什麼、該怎麼處置。任務導向、資訊密度優先，不是瀏覽式閱讀。
2. **實際使用者（本專案的真實情境）**：檢視這份求職作品集的技術面試官／招募方。他們會在螢幕錄影或現場 demo 中看到這個介面，用它判斷作者的系統整合與 API 工程能力。介面必須讓「這是一個認真的工程工具」這件事一眼可信，而不是一個貼皮的範例腳本。

## Product Purpose

本地 YOLO26 ONNX 偵測 + 商用 VLM API 的兩段式巡檢報告 pipeline 之操作介面：拖入影像、選供應商/模型、觸發批次、檢視標註圖與逐圖評估、下載報告。成功的樣子：面試官錄的 15–20 秒 demo GIF 一看就懂架構在做什麼，介面本身傳達出跟後端程式碼同等的工程嚴謹度。

## Brand Personality

沉穩、精確、克制。深色為主的工具介面，中性灰階承載大部分畫面，強調色只用在少數互動節點（按鈕、連結、進度指示），不用來裝飾。個性更接近企業內部監控/維運系統，而非消費性 SaaS 產品——低調、可信、不搶戲。

## Anti-references

- 常見「AI 生成」介面老套：漸層文字、卡片轟炸（同尺寸卡片重複排列）、預設 Gradio 橘色主按鈕、暖色調米白/沙色背景、小型全大寫追蹤字距的 eyebrow 標籤、01/02/03 編號裝飾。
- 玻璃擬態（glassmorphism）當裝飾用。
- 圓角 24px 以上的卡片/輸入框。
- 強調色跟報告內部的嚴重度色階（重大=紅、中等=橘、輕微=灰黃）混用——兩者語義不同，介面強調色必須是獨立的品牌色。

## Design Principles

- **資訊密度優先於留白美感**：這是給人快速掃描判斷用的巡檢工具，不是行銷頁；排版服務於「一眼看出哪張板子有問題」。
- **深色主題，但可讀性優先**：所有文字對比必須達 WCAG AA（本文 ≥4.5:1），深色不能犧牲報告表格與標註圖的可辨識度。
- **強調色獨立於語義色**：互動元件（按鈕/連結/進度條）用單一品牌強調色；嚴重度紅/橘/灰黃只存在於報告內容語義中，兩套顏色不互相借用。
- **克制的網域呼應**：可以讓配色隱約呼應 PCB／電路板的視覺語言（如深綠、金屬感），但不做成裝飾性插圖或電路紋理背景。
- **狀態清楚可見**：上傳、偵測中、產出報告、完成下載，每個階段都要有清楚的視覺回饋——這直接影響 demo 錄影的可讀性。

## Accessibility & Inclusion

WCAG AA 對比為底線（本文 ≥4.5:1、大字/粗體 ≥3:1）。無特殊使用者需求，但任何新增動畫需提供 `prefers-reduced-motion` 的降級（瞬時切換或淡入淡出）。
